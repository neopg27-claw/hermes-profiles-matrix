Profiles turn Hermes into a crew.

A Hermes profile is a separate Hermes home directory with its own config, credentials, memory, sessions, skills, cron jobs, and gateway state. Profiles keep operating context clean, but they are not security sandboxes: on the local terminal backend, each profile still has the same filesystem access as the OS user.

What profiles isolate

Different shells. Different memories. Same agent engine.

Profiles keep context boundaries clean: a life-admin profile can carry preferences about travel and finance, a health profile can remember training constraints, and a research profile can optimize for evidence. You avoid one giant persona accumulating conflicting habits.

Recommended operating model

Neo remains the operator-of-record.

Keep Neo as the default companion, orchestrator, and generalist. The siblings are not hard-walled silos. They are better-fit contexts with soft handoff rules, shared taste, and domain-specific memory.

  • Tech, coding, automation, and agent setup stay with Neo.
  • Life/admin, health, and research route to the sibling when that improves quality.
  • Prefer soft behavioral rules over restrictive tool access.

Research-backed details

What Hermes profiles actually isolate.

In Hermes, a profile is a separate Hermes home directory. That makes profiles useful for running Neo, Trinity, Morpheus, and Oracle as distinct operating contexts without mixing state.

TopicWhat to know
ProfileA separate Hermes home directory for one agent context.
Isolated stateEach profile can have its own config.yaml, .env, SOUL.md, memory, sessions, skills, cron jobs, state database, gateway state, logs, and PID files.
Not isolatedProfiles are not sandboxes. With the local terminal backend, the agent has the same filesystem access as the user account.
Creation modesCreate a blank profile, clone config/personality with --clone, clone everything with --clone-all, or clone from another profile with --clone-from.
UsageUse hermes -p oracle, set a sticky default with hermes profile use oracle, or run profile aliases like oracle chat once created.
GatewayThe messaging gateway connects platform adapters, keeps per-chat sessions, runs cron jobs, routes messages to the agent, and delivers replies.
MemoryMemory is bounded and curated. It is injected into the system prompt at session start, then changes appear in the prompt on the next session.
Matrix crewNeo, Trinity, Morpheus, and Oracle are Pranjal's local profile conventions, not built-in Hermes defaults.
The active crew
Trinity

Life, admin, logistics

Use for personal admin, finance, taxes, visa, shopping, travel, household logistics, and the thousand small loops that should not pollute the coding context.

  • Tools: Gmail, Calendar, Drive, Notion, web
  • Style: organized, direct, permission-aware
  • Boundary: creates/edits are okay where allowed; deletion and sending require approval
profile: trinity // mode: life ops
Morpheus

Health, fitness, recovery

Use for nutrition, workouts, symptoms, doctor prep, habit loops, recovery plans, and keeping health context separate from general operations.

  • Tools: notes, calendar, web, memory
  • Style: grounded, cautious, pattern-aware
  • Boundary: supports decisions; does not pretend to be a doctor
profile: morpheus // mode: body signal
The Oracle

Research, evidence, synthesis

Use for deep research, source-checking, market scans, paper trails, literature reviews, and separating what is known from what is merely plausible.

  • Tools: web, arXiv, docs, notes, citations
  • Style: evidence-first, calm, caveated
  • Boundary: no fake certainty; source quality is part of the answer
profile: oracle // mode: signal extraction

Configuration advice

Soft handoffs beat hard cages.

Neo owns tech, coding, automation, and the front-door experience. Trinity owns life/admin. Morpheus owns health/fitness. Oracle owns research. Keep tools broad enough to be useful, but encode domain boundaries in SOUL.md and AGENTS.md so each profile knows when to answer and when to point elsewhere.
